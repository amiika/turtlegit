What!?

turtlegit is a Linked Data endpoint backed by local git repository that stores graphs as ordered turtle (.ttl) files.

Supports SPARQL Graph HTTP Protocol (GET / PUT / POST) with different RDF content-types, but everything is eventually serialized as ordered turtle.

Why?

Sometimes provenance is more important than speed.

Enables collaborative editing with conflict resolution.

How to use it?

Simply GET / POST / PUT http://127.0.0.1:5000/{myRepoDir}?graph=file:///etc/{workspace}/{myRepoDir}/myTurtleFile.ttl

Configuration

Configure your workspace directory to settings.ini

Create git repository under workspace using git init. etc

Add some ttl files. Wonder.

Future milestones
